## Design Context

### Users
Primary user is a solo developer working locally inside a focused execution loop. They are using Superplan while actively coding, reviewing task state, and trying to stay in flow without opening a heavier interface unless necessary. The overlay exists to provide a fast status glance and only earn more attention when something actually matters.

### Brand Personality
The product should feel calm, quiet, and controlled. It should read as trustworthy, local-first, and execution-oriented rather than playful, flashy, or dashboard-like. The interface should support concentration, reduce drift, and feel like a disciplined companion rather than a noisy assistant.

### Aesthetic Direction
Follow the existing desktop theme rather than inventing a disconnected visual language. Use restrained contrast, muted surfaces, and a utility-focused composition that still feels deliberate and premium. Avoid anything that reads like a macOS widget clone, decorative glassmorphism, or generic AI-generated dark card UI.

### Design Principles
1. Reduce interruption: the overlay should be scannable in seconds and never feel like a second dashboard.
2. Preserve flow: emphasize the one thing that matters now, then provide a clean path to the full board.
3. Stay visually quiet: use hierarchy, spacing, and type before decorative effects.
4. Match the host product: overlay chrome should feel like an extension of the existing desktop app, not a new brand.
5. Make urgency legible: feedback-needed and blocked states should stand out through structure and wording, not visual noise.
